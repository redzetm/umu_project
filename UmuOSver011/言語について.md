# C言語について

UmuOS の実装において、私は C 言語を好んで使用している。

これは流行や性能比較の結果ではなく、  
C 言語が OS や Linux カーネルの挙動を理解するための  
最も直接的な言語だと感じているからである。

C 言語は多くを隠さない。

- メモリは明示的に扱う必要がある
- 失敗は戻り値として現れる
- 抽象化は最小限である

その不親切さこそが、  
OS の内部構造を観測し、理解する上で重要だと考えている。

UmuOS において C 言語は「唯一の正解」ではない。  
将来、別の言語を試す可能性も否定しない。

それでも現時点では、  
C 言語が最も思考と実装の距離が近い言語であり、  
UmuOS の研究目的に最も適していると判断している。

この選択は思想ではなく、  
その時点での最良の道具を選んだ結果である。
