UmuOS User Shell (ush) — 仕様書（0.0.7）  
Target OS: UmuOS 系（少なくとも 0.1.7-base-stable 系で成立することを狙う）

この文書は ush 0.0.6 の仕様を引き継ぎつつ、ush 0.0.7 で追加・変更する内容を利用者視点でまとめた仕様書である。

注意:

- 実装の唯一の正は [ush-0.0.7-詳細設計書.md](ush-0.0.7-詳細設計書.md) とする。

# 1. 目的と非目的

## 1.1 目的

- UmuOS におけるユーザー向け対話シェルとして、軽量で観測可能な操作環境を提供する。
- 0.0.6 までの小物スクリプト機能を維持しつつ、0.0.7 では対話コマンドラインで UTF-8 日本語を壊さず扱えるようにする。
- brace expansion の最小サブセットを追加し、日常的な複数引数展開を ush で扱えるようにする。

## 1.2 非目的

- POSIX sh / bash 完全互換
- locale/IME を含む総合的な日本語入力環境
- ジョブ制御、関数、算術式、配列
- 高度な FD 操作、ヒアドキュメント、多段パイプ
- bash 完全互換の brace expansion

複雑な sh 互換スクリプトが必要な場合は `/bin/sh`（BusyBox ash）を使う。

# 4. 機能一覧（0.0.7）

## 4.1 対話シェル

- プロンプト表示
- 行編集（最低限）
  - Backspace
  - Delete
  - 左右矢印
  - 上下矢印による履歴移動
  - Tab 補完（簡易）
- UTF-8 日本語を含む入力の受け付け
- PATH 検索（未設定時は `/umu_bin:/sbin:/bin`）
- 外部コマンド実行（fork/exec）
- builtins
  - `cd`, `pwd`, `export`, `test`, `[`, `exit`, `help`

## 4.2 シェル言語

### 0.0.6 までの継承機能

- 演算子: `|`（1 段のみ）, `&&`, `||`, `;`, `<`, `>`, `>>`
- クォート: `'...'`, `"..."`
- 展開: `$VAR`, `${VAR}`, `$?`, `~`
- 位置パラメータ: `$0`, `$1`..`$9`, `$#`
- コマンド置換: `$(...)`
- コメント: トークン先頭 `#` から行末
- 複数行ブロック
- 制御構文
  - `if/then/elif/else/fi`
  - `while/do/done`
  - `for NAME in WORDS... do ... done`
  - `case WORD in PAT) ... ;; ... esac`

### 0.0.7 で追加・強化する機能

- 対話コマンドラインで UTF-8 日本語を壊さず扱う
- unquoted WORD に対する最小限の brace expansion

# 5. 対話入力仕様

## 5.1 日本語入力

- UTF-8 日本語を含む入力を受け付ける。
- 日本語を含むコマンド名、引数、ファイル名を byte 列として保持し、そのまま tokenize / expand / exec へ渡す。

## 5.2 カーソル移動と削除

- 左右矢印は UTF-8 1 文字単位で移動する。
- Backspace はカーソル左の UTF-8 1 文字を削除する。
- Delete はカーソル位置の UTF-8 1 文字を削除する。

## 5.3 表示の前提

- 表示幅は簡易計算とする。
- 非 ASCII の表示幅は簡易に 2 とみなす。
- そのため、端末や文字種によっては厳密な見た目一致を保証しない。
- ただし、0.0.7 の目標は「壊さないこと」であり、「完全な Unicode 端末編集器」ではない。

---

# 6. 入力単位（複数行ブロック）

ush は入力を 1 行単位ではなく、完結したブロック単位で評価する。

- 対話モード
  - `fi` / `done` / `esac` などでブロックが閉じるまで追加入力を受け付ける。
- スクリプトモード
  - ファイルを上から読み、ブロックが完結した時点で評価する。
- EOF でブロックが未完結なら `syntax error` とする。

---

# 7. 字句仕様（tokenize）

## 7.1 区切り

- 空白（スペース/タブ）で区切る。
- 演算子は空白の有無に関わらず独立トークンとして認識する。
- 非 ASCII バイトは特別扱いせず、空白や演算子でなければ WORD の一部として扱う。

## 7.2 演算子トークン

- `|`, `&&`, `||`, `;`, `<`, `>`, `>>`, `;;`, `)`

備考:

- `)` は `case` の `PAT)` 用である。
- `;;` は `case` の項目終端である。

## 7.3 コメント

- 未クォートで、かつトークン先頭の `#` をコメント開始として扱う。

## 7.4 波括弧の扱い

- `${VAR}` のための `{` と brace expansion の `{` は tokenize 時点では区別しない。
- unquoted WORD の先頭に `{` が来ても、0.0.7 では即エラーにせず WORD として保持する。

# 8. 展開仕様

## 8.1 通常の word 展開

次を扱う。

- `$VAR`
- `${VAR}`
- `$?`
- `$0`, `$1`..`$9`, `$#`
- `~`
- `$(...)`

展開結果は UTF-8 文字列であっても byte 列として次段へ渡す。

## 8.2 brace expansion

0.0.7 で対応するのは最小サブセットのみである。

対応例:

- `cp {aaa,bbb,ccc,ddd} /umu_bin/`
- `cp foo{1,2}.txt /tmp/`
- `echo /home/{tama,root}/tmp`

成立条件:

- 対象は unquoted WORD のみ
- 1 語につき 1 組の brace pair のみ
- prefix / suffix を許可
- 各要素はカンマ区切り
- 空要素は許可しない

未対応:

- nested brace expansion
- range 形式
- quoted brace expansion
- 1 語内の複数 brace pair の直積展開

## 8.3 展開順序

0.0.7 の展開順序は次の通りとする。

1. brace expansion
2. 通常の word 展開
3. glob

この順序により、`${VAR}` は従来通り使え、変数展開結果に含まれる `{A,B}` を brace expansion として再解釈しない。

# 9. 構文概要

## 9.1 if

```sh
if <cond>
then
  <then-body>
elif <cond>
then
  <elif-body>
else
  <else-body>
fi
```

## 9.2 while

```sh
while <cond>
do
  <body>
done
```

## 9.3 for

```sh
for NAME in WORDS...
do
  <body>
done
```

## 9.4 case

```sh
case WORD in
  PAT) <body> ;;
  PAT2|PAT3) <body> ;;
  *) <body> ;;
esac
```

# 10. 条件評価 builtin（test / [ ]）

`test EXPR` と `[ EXPR ]` を builtin として提供する。

- 成功（真）: 0
- 失敗（偽）: 1
- 構文/引数不正: 2

サポートする最小サブセット:

- 単項: `-n STR`, `-z STR`, `-e PATH`, `-f PATH`, `-d PATH`
- 二項（文字列）: `A = B`, `A != B`
- 否定: `! EXPR`
- 単一引数: `STR`（空でなければ真）

---

# 11. 実行モデル

- ush は `last_status` を保持する。
- `$?` は `last_status` を返す。
- builtins
  - 成功: 0
  - 実行エラー: 1
  - 構文/引数不正: 2
- 外部コマンド探索
  - 未発見: 127
  - 実行不可: 126
- `ENOEXEC` のときのみ `/bin/sh` へフォールバックする。

---

# 12. 未対応（検出してエラー、または対象外）

- 単体 `&`
- 行継続（`\` + 改行）
- ヒアドキュメント（`<<` / `<<<`）
- 多段パイプ（`a|b|c`）
- `(` 単体（`$(...)` を除く）
- nested なコマンド置換
- 関数、算術式、配列
- `break` / `continue`
- brace expansion の未対応形式
  - nested
  - range
  - quoted
  - 複数 brace pair の直積

---

# 13. 受け入れ条件

少なくとも次を満たすことを 0.0.7 の受け入れ条件とする。

- `printf '%s\n' 日本語abc` のような未クォート日本語引数が壊れない
- 日本語ファイル名へのリダイレクトができる
- 対話入力で左右移動、Backspace、Delete が UTF-8 1 文字単位で動く
- `printf '%s\n' {aaa,bbb}` が複数行へ展開される
- `X='{A,B}'; printf '%s\n' "${X}"` が `{A,B}` のまま出力される

---

# 14. まとめ

ush 0.0.7 は、0.0.6 の軽量シェル路線を維持しながら、対話入力の実用性を大きく改善する版である。

利用者視点で重要なのは次の 2 点である。

- 日本語を含むコマンドラインを ush 自身で扱えること
- brace expansion を最小限だけ利用できること

一方で、完全な sh 互換や完全な Unicode 編集器を目指す版ではない点は、引き続き明確にしておく。
