# 詳細設計書-0.1.3（手順：同時ログイン / ttyS1=TCPシリアル）

この文書は、UmuOS-0.1.3 を「最初から再構築できるベース」として成立させるための作業手順を、コマンドレベルでまとめる。

## 重要方針

- 0.1.3 の主目的は **同時ログイン（ttyS0 + ttyS1）**。
- 0.1.3 では `su` とログ基盤は扱わない（0.1.4以降）。
- ttyS1 は PTY ではなく **QEMUのTCPシリアル** を使う。

> NOTE: `telnet` を使うが、これは「ゲストtelnetd」ではなく、ホストが QEMU の TCP ポートへ接続しているだけ。

---

## 0. 固定パラメータ（この文書で固定する値）

- 作業ルート：`~/umu/umu_project/UmuOS-0.1.3`
- Kernel version：`6.18.1`
- ISO 名：`UmuOS-0.1.3-boot.iso`
- initrd 名：`initrd.img-6.18.1`
- kernel 名：`vmlinuz-6.18.1`
- 永続ディスク：`disk/disk.img`（ext4、4GiB、UUIDでroot指定）
- QEMU ディスク：virtio-blk（`if=virtio`）
- initramfs `/init`：musl 静的リンク（`musl-gcc -static`）

ttyS1 のポート（推奨固定）：`TTYS1_PORT=5555`

## 6. initramfs（/init）作成

initramfs の責務は「root=UUID の ext4 を mount して switch_root」まで。

### 6.1 `/init` のソース作成

```bash
cat > ~/umu/umu_project/UmuOS-0.1.3/initramfs/src/init.c <<'EOF'
#define _GNU_SOURCE
#include <errno.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <sys/mount.h>
#include <sys/stat.h>
#include <sys/types.h>
#include <unistd.h>

static void must(int rc, const char *what) {
  if (rc == 0) return;
  fprintf(stderr, "[init] %s: %s\n", what, strerror(errno));
  for (;;) sleep(1);
}

int main(int argc, char **argv) {
  (void)argc; (void)argv;

  mkdir("/proc", 0555);
  mkdir("/sys", 0555);
  mkdir("/dev", 0555);
  mkdir("/newroot", 0755);

  must(mount("proc", "/proc", "proc", 0, NULL), "mount proc");
  must(mount("sysfs", "/sys", "sysfs", 0, NULL), "mount sysfs");
  must(mount("devtmpfs", "/dev", "devtmpfs", 0, NULL), "mount devtmpfs");

  // root=UUID は kernel cmdline で渡す（GRUB）。
  // /dev/vda と決め打ちせず、/dev/disk/by-uuid/<UUID> を使う。
  const char *uuid = getenv("DISK_UUID");
  if (!uuid || !*uuid) {
    fprintf(stderr, "[init] DISK_UUID env is missing\n");
    for (;;) sleep(1);
  }
  char devpath[256];
  snprintf(devpath, sizeof(devpath), "/dev/disk/by-uuid/%s", uuid);

  must(mount(devpath, "/newroot", "ext4", MS_RDONLY, NULL), "mount newroot(ro)");
  must(mount(devpath, "/newroot", "ext4", 0, NULL), "mount newroot(rw)");

  must(chdir("/newroot"), "chdir /newroot");
  must(mount("/dev", "/newroot/dev", NULL, MS_MOVE, NULL), "move /dev");
  must(mount("/proc", "/newroot/proc", NULL, MS_MOVE, NULL), "move /proc");
  must(mount("/sys", "/newroot/sys", NULL, MS_MOVE, NULL), "move /sys");

  must(chroot("."), "chroot");
  must(chdir("/"), "chdir /");

  char *const init_argv[] = { "/sbin/init", NULL };
  execv("/sbin/init", init_argv);
  fprintf(stderr, "[init] exec /sbin/init failed: %s\n", strerror(errno));
  for (;;) sleep(1);
}
EOF
```

## 9. QEMU 起動（ttyS1 を TCP で公開）

OVMF_VARS は書き換えが入るためローカルにコピーする。

```bash
cp -f /usr/share/OVMF/OVMF_VARS_4M.fd \
  ~/umu/umu_project/UmuOS-0.1.3/run/OVMF_VARS.fd
```

起動（ttyS0=stdio、ttyS1=tcp）：

```bash
TTYS1_PORT=5555
echo "[host] ttyS1 tcp: 127.0.0.1:${TTYS1_PORT}"

DISK_UUID="$DISK_UUID" \
qemu-system-x86_64 \
  -machine q35,accel=tcg \
  -m 1024 \
  -drive if=pflash,format=raw,readonly=on,file=/usr/share/OVMF/OVMF_CODE_4M.fd \
  -drive if=pflash,format=raw,file=~/umu/umu_project/UmuOS-0.1.3/run/OVMF_VARS.fd \
  -cdrom ~/umu/umu_project/UmuOS-0.1.3/UmuOS-0.1.3-boot.iso \
  -drive file=~/umu/umu_project/UmuOS-0.1.3/disk/disk.img,if=virtio,format=raw \
  -serial stdio \
  -serial tcp:127.0.0.1:${TTYS1_PORT},server,nowait \
  -no-reboot
```

### 6.7 rcS（初期化 + ②boot.log 追記）

```bash
sudo tee /mnt/umuos013/etc/init.d/rcS >/dev/null <<'EOF'
#!/bin/sh

# 失敗しても止めない（観測を優先）
mount -t proc proc /proc 2>/dev/null || true
mount -t sysfs sysfs /sys 2>/dev/null || true
mount -t devtmpfs devtmpfs /dev 2>/dev/null || true
mount -t devpts devpts /dev/pts 2>/dev/null || true

# ② 永続ログ（/logs/boot.log）
(
  mkdir -p /logs

  UPTIME_S="$(cut -d' ' -f1 /proc/uptime 2>/dev/null || echo '?')"
  BOOT_ID="$(cat /proc/sys/kernel/random/boot_id 2>/dev/null || echo '?')"

  {
    echo ""
    echo "===== boot begin (boot_id=${BOOT_ID} uptime_s=${UPTIME_S}) ====="
    echo "[cmdline]"
    cat /proc/cmdline 2>/dev/null || true
    echo ""
    echo "[mount]"
    mount 2>/dev/null || true
    echo ""
    echo "[dmesg]"
    dmesg 2>/dev/null || true
    echo "===== boot end ====="
  } >> /logs/boot.log
) 2>/dev/null || (
  /bin/echo "[rcS] WARN: boot.log write failed" > /dev/console 2>/dev/null || true
)

/bin/echo "[rcS] rcS done" > /dev/console 2>/dev/null || true
EOF

sudo chmod 755 /mnt/umuos013/etc/init.d/rcS
```

### 6.9 `su -` を安全寄りに成立させる（0.1.3差分：案A）

ただし BusyBox 全体を setuid にすると権限が広がりやすいので、`su` 専用の実体（`busybox.su`）を作って setuid を限定する。

注意：`busybox --install` により `/bin/su -> /bin/busybox` が作られている可能性があるため、**applet 生成の後**に差し替える。

```bash
# su 専用 BusyBox を作る
sudo cp -a /mnt/umuos013/bin/busybox /mnt/umuos013/bin/busybox.su
sudo chown root:root /mnt/umuos013/bin/busybox.su
sudo chmod 4755 /mnt/umuos013/bin/busybox.su

# /bin/su を su専用 BusyBox に向ける
sudo ln -sf /bin/busybox.su /mnt/umuos013/bin/su

# 確認
sudo ls -l /mnt/umuos013/bin/busybox /mnt/umuos013/bin/busybox.su /mnt/umuos013/bin/su
```

期待（例）：

- `/mnt/umuos013/bin/busybox` は `-rwxr-xr-x`
- `/mnt/umuos013/bin/busybox.su` は `-rwsr-xr-x`

## 7. initramfs（/init）作成（musl 静的）

### 7.1 rootfs 骨格（initramfs側）

```bash
cd ~/umu/umu_project/UmuOS-0.1.3/initramfs
rm -rf rootfs
mkdir -p rootfs/{bin,sbin,etc,proc,sys,dev,dev/pts,run,newroot}

cp /bin/busybox rootfs/bin/busybox
chmod 755 rootfs/bin/busybox

# 混線防止：BusyBox applet の生成は chroot して実行する
sudo chroot rootfs /bin/busybox --install -s /bin

# switch_root を必ず用意（BusyBox applet）

```

## 10. ① 開発環境側ログ（QEMU シリアル出力の保存）を含む起動スクリプト

方針：

- 保存先は `logs/` に固定
- 命名は `qemu-serial_YYYYmmdd_HHMMSS.log`
- 端末表示は維持（ログイン操作に必要）
- `script(1)` を使い ttyS0（stdio）のセッションを記録する
- 0.1.3 差分：ttyS1 用に `-serial pty` を追加し、別端末から接続して同時ログインする

ttyS1 もログ取得を必須にする場合、ttyS1 に接続する側の端末を `script(1)` で包んで別ファイルに保存する。

### 10.1 umuOSstart.sh 作成

```bash
cd ~/umu/umu_project/UmuOS-0.1.3
tee umuOSstart.sh >/dev/null <<'EOF'
#!/usr/bin/env bash
set -euo pipefail

LOG_DIR=~/umu/umu_project/UmuOS-0.1.3/logs
RUN_DIR=~/umu/umu_project/UmuOS-0.1.3/run
ISO=~/umu/umu_project/UmuOS-0.1.3/UmuOS-0.1.3-boot.iso
DISK=~/umu/umu_project/UmuOS-0.1.3/disk/disk.img

mkdir -p "${LOG_DIR}" "${RUN_DIR}"

TS="$(date +%Y%m%d_%H%M%S)"
LOG_FILE="${LOG_DIR}/qemu-serial_${TS}.log"

echo "[umuOSstart] log: ${LOG_FILE}"

echo "[umuOSstart] NOTE: ttyS1 is provided as a PTY."
echo "[umuOSstart] NOTE: QEMU will print something like: char device redirected to /dev/pts/N"

OVMF_CODE="/usr/share/OVMF/OVMF_CODE_4M.fd"
OVMF_VARS_SRC="/usr/share/OVMF/OVMF_VARS_4M.fd"
OVMF_VARS="${RUN_DIR}/OVMF_VARS_umuos_0_1_3.fd"

if [[ ! -f "${OVMF_VARS}" ]]; then
  cp -f "${OVMF_VARS_SRC}" "${OVMF_VARS}"
fi

QEMU_CMD=(
  qemu-system-x86_64
  -machine q35,accel=tcg
  -m 1024
  -cpu qemu64
  -drive if=pflash,format=raw,readonly=on,file="${OVMF_CODE}"
  -drive if=pflash,format=raw,file="${OVMF_VARS}"
  -cdrom "${ISO}"
  -drive file="${DISK}",format=raw,if=virtio
  -nographic
  -serial stdio
  -serial pty
  -monitor none
)

exec script -q -f --command "$(printf '%q ' "${QEMU_CMD[@]}")" "${LOG_FILE}"
EOF

chmod 755 umuOSstart.sh
```

### 10.2 ttyS1 接続スクリプト（ログ取得用：0.1.3差分）

ttyS1（`/dev/pts/N`）への接続とログ保存をワンコマンド化する。

```bash
cd ~/umu/umu_project/UmuOS-0.1.3
tee connect_ttyS1.sh >/dev/null <<'EOF'
#!/usr/bin/env bash
set -euo pipefail

if [[ $# -ne 1 ]]; then
  echo "Usage: $0 /dev/pts/N" >&2
  exit 2
fi

PTY="$1"
LOG_DIR=~/umu/umu_project/UmuOS-0.1.3/logs

mkdir -p "${LOG_DIR}"

TS="$(date +%Y%m%d_%H%M%S)"
LOG_FILE="${LOG_DIR}/ttyS1_${TS}.log"

echo "[connect_ttyS1] pty: ${PTY}"
echo "[connect_ttyS1] log: ${LOG_FILE}"

echo "[connect_ttyS1] NOTE: This logs the ttyS1 session (screen + script)."
exec script -q -f --command "screen ${PTY} 115200" "${LOG_FILE}"
EOF

chmod 755 connect_ttyS1.sh
```

### 11.2 ttyS1（2本目シリアル）へ接続

起動直後、ttyS0 側（起動した端末）に QEMU から以下のような行が出る。

```bash

```

`login:` が出れば ttyS1 が成立している。

## 13. トラブルシュート（0.1.3の追加観点）

### 13.1 ttyS1 に `login:` が出ない

- ext4 側の `/etc/inittab` に `ttyS1::respawn:...` が入っているか
- QEMU 起動オプションに `-serial pty` が入っているか
- どの PTY に割り当てられたか（ttyS0 側に出る `char device redirected ...` を確認）

### 13.2 `su: must be suid to work properly`

- `/bin/su -> /bin/busybox.su` になっているか
- `/bin/busybox.su` が `-rwsr-xr-x`（setuid）になっているか
- `/` が `nosuid` でマウントされていないか（`mount | grep ' on / '`）
