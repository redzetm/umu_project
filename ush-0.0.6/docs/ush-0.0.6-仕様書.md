# ush-0.0.6-仕様書.md
UmuOS User Shell (ush) — 仕様書（0.0.6）  
Target OS: UmuOS-0.1.7-base-stable（想定）

この文書は、ush 0.0.5 の仕様（`$(...)` と位置パラメータ）を引き継ぎつつ、ush 0.0.6 で **複数行ブロック** と **制御構文** と **条件評価 builtin** を追加した仕様をまとめる。

注意（このリポジトリ運用）:
- 実装の唯一の正は [ush-0.0.6-詳細設計書.md](ush-0.0.6-詳細設計書.md) とする。
- 本書は仕様（要約）であり、矛盾がある場合は詳細設計書を優先する。

---

# 1. 目的と非目的

## 1.1 目的
- UmuOS におけるユーザー向け対話シェルとして、軽量で観測可能な操作環境を提供する。
- POSIX 互換は目的にしない（ただし挙動は曖昧にせず仕様で固定）。
- `/bin/sh`（BusyBox ash）へ逃げずに、以下の「小物スクリプト」までを ush で書けるようにする。
  - `&&` / `||` と `;` による直列実行
  - `$(...)` と位置パラメータ
  - `if/while/for/case` と `test/[ ]` によるブロック制御

## 1.2 非目的（0.0.6でもやらない）
- POSIX sh / bash 互換の追求
- ジョブ制御（バックグラウンド実行 `&`、fg/bg、`jobs` 等）
- 関数、算術式、配列
- `break` / `continue`
- ヒアドキュメント（`<<` / `<<<`）
- 高度な FD 操作（例: `2>file`、`2>&1`、`n>file` 等）
- 多段パイプ（`a|b|c`）
- コマンド置換の入れ子 `$( $(...) )`

本格スクリプトが必要な場合は `/bin/sh`（BusyBox ash）で実行する。

## 2.1 役割分担
- `/umu_bin/ush` : 対話操作・軽量スクリプト（本仕様）
- `/bin/sh` : 互換性重視のスクリプト（BusyBox ash）

- `musl-gcc -static` による静的リンクを基本とする。

# 4. 機能一覧（0.0.6）

## 4.1 対話シェル
- プロンプト表示
- 行編集（最低限）
  - Backspace/Delete/左右矢印/上下（履歴）
  - Tab 補完（簡易）
- PATH 検索（未設定時は `/umu_bin:/sbin:/bin`）
- 外部コマンド実行（fork/exec）
- builtins:
  - `cd`, `pwd`, `export`, `test`, `[`, `exit`, `help`

## 4.2 シェル言語（小物スクリプト向け）
### 0.0.5 まで（継承）
- 演算子: `|`（1段のみ）, `&&`, `||`, `;`, `<`, `>`, `>>`
- クォート: `'...'`, `"..."`
- 展開: `$VAR`, `${VAR}`, `$?`, `~`
- 位置パラメータ: `$0`, `$1`..`$9`, `$#`
- コマンド置換: `$(...)`（stdout 捕捉）
- コメント: トークン先頭 `#` から行末

### 0.0.6 で追加
- 複数行ブロック（対話/スクリプト）
- 制御構文
  - `if/then/elif/else/fi`
  - `while/do/done`
  - `for NAME in WORDS... do ... done`
  - `case WORD in PAT) ... ;; ... esac`

# 5. 入力単位（複数行ブロック）

ush は入力を「1行」ではなく「完結したブロック単位」で評価する。

- 対話モード: `fi` / `done` / `esac` 等でブロックが閉じるまで、追加の行を読み続ける。
- スクリプトモード: ファイルを上から読み、ブロックが完結したタイミングで評価する。
- ブロックが EOF で閉じなかった場合は `syntax error`（`last_status=2`）。
- 1ブロックの最大長は `USH_MAX_LINE_LEN` に依存し、超過は `syntax error`（`last_status=2`）。

# 6. 字句（tokenize）

## 6.1 区切り
- 空白（スペース/タブ）で区切る。
- 演算子は空白の有無に関わらず独立トークンとして認識する。

## 6.2 演算子トークン
- `|`, `&&`, `||`, `;`, `<`, `>`, `>>`, `;;`, `)`

備考:
- `)` は `case` の `PAT)` 用。
- `;;` は `case` の項目終端。

## 6.3 コメント
- 未クォートで、かつ「トークン先頭の `#`」をコメント開始として扱う。

# 7. 構文（制御構文の概要）

## 7.1 if
```
if <cond>
then
  <then-body>
elif <cond>
then
  <elif-body>
else
  <else-body>
fi
```

## 7.2 while
```
while <cond>
do
  <body>
done
```

## 7.3 for（語リスト）
```
for NAME in WORDS...
do
  <body>
done
```

## 7.4 case
```
case WORD in
  PAT) <body> ;;
  PAT2|PAT3) <body> ;;
  *) <body> ;;
esac
```

# 8. 条件評価 builtin（test / [ ]）

`test EXPR` と `[ EXPR ]` を builtin として提供する。

- 成功（真）: 0
- 失敗（偽）: 1
- 構文/引数不正: 2

サポートする EXPR（最小サブセット）:
- 単項: `-n STR`, `-z STR`, `-e PATH`, `-f PATH`, `-d PATH`
- 二項（文字列）: `A = B`, `A != B`
- 否定: `! EXPR`
- 単一引数: `STR`（空でなければ真）

---

# 9. 実行モデル（last_status）

- ush は `last_status`（初期値 0）を保持する。
- builtins:
  - 成功: 0
  - 実行エラー: 1
  - 構文/引数不正: 2
- 外部コマンド探索:
  - 未発見: 127
  - 実行不可（権限等）: 126

---

# 10. 未対応（検出してエラー）

- 単体 `&`
- 未クォートの `{` `}`（`${NAME}` を除く）
- `(`（`$(...)` を除く）
- 行継続（`\` + 改行）
- ヒアドキュメント（`<<` / `<<<`）
- 多段パイプ（`a|b|c`）
