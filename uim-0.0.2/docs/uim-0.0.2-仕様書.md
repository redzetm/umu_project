UmuOS Interactive Minimal editor (uim) — 仕様書（0.0.2）

# 1. 目的と非目的

## 1.1 目的
- UmuOS 上で「コードを読みながら作業する」ための最小エディタを提供する
- vim の代替ではなく、最低限の vi ライク操作を再現する
  - NORMAL / INSERT / `:`（コロン）
  - `dd` / `yy` / `p` / `a`
  - `/` / `n`
  - `:w` / `:q` / `:wq` / `:q!`
- 日本語コメント（UTF-8）を壊さずに編集できる
- 初期版から「全角幅がそれなりに正確」なカーソル表示を実現する

## 1.2 非目的（0.0.2 ではやらない）
- vim/vi 互換の追求（完全互換はしない）
- 置換、undo/redo、ビジュアルモード、複数バッファ、マクロ等
- 構文ハイライト、補完、LSP 等の IDE 的機能
- 厳密な Unicode 幅計算（East Asian Width の完全実装など）
- 行内削除（`x`）や単語移動（`w`/`b`）などの追加コマンド
- `?`（後方検索）や `N`（逆方向）、正規表現検索、インクリメンタル検索、検索ハイライト

# 2. 実行環境と成果物

- 成果物: 単一バイナリ（例: `uim`）
- 端末: ANSI エスケープを解釈できる端末を想定
- 文字: 端末は UTF-8 表示できるものを想定

---

# 3. 機能一覧

## 3.1 起動
- `uim <file>` で指定ファイルを編集する
- ファイルが存在しない場合は空のバッファとして開始できる
- `--batch` を指定すると、標準入力から疑似キー入力を受け取って非対話実行できる（スモークテスト用）
- `--version` を指定すると、バージョン文字列（例: `0.0.2`）を標準出力に出して終了する

## 3.2 モード
- NORMAL: 移動、`0`/`$`、`dd`/`yy`/`p`、`:`、`/`、`n`、`i`、`a`（一部コマンドは数値プレフィクス対応）
- INSERT: 文字挿入、Enter による改行、Backspace による 1 文字削除、矢印キーで移動、ESC で NORMAL へ
- COLON（`:`）: コマンド入力、Enter で実行、ESC でキャンセル
- SEARCH（`/`）: 検索文字列入力、Enter で検索、ESC でキャンセル

## 3.3 移動
- `h` 左 / `j` 下 / `k` 上 / `l` 右
- 矢印キーも対応（内部的に `h/j/k/l` と同等として扱う）
- `0` : 行頭へ移動（※数値プレフィクスが未入力で、かつペンディングコマンドが無い場合のみ）
- `$` : 行末へ移動
- `gg` / `Ngg` : 先頭行 / N 行目へ移動（例: `gg`, `10gg`）
- `G` / `NG` : 最終行 / N 行目へ移動（例: `G`, `10G`）
- Home/End キー: 行頭/行末へ移動（`0`/`$` と同等の意図）

## 3.4 行操作（NORMAL）
- `dd` : 現在行を削除し、削除した行をヤンクバッファへ入れる
- `Ndd` : 現在行から N 行削除し、削除した N 行をヤンクバッファへ入れる（例: `3dd`）
- `yy` : 現在行をヤンクバッファへ入れる
- `Nyy` : 現在行から N 行をヤンクバッファへ入れる（例: `3yy`）
- `p`  : ヤンクバッファの内容を「現在行の下」に貼り付ける（ヤンクが複数行の場合は複数行を挿入する）
- `a`  : カーソルの 1 文字右に移動して INSERT に入る（行末ではそのまま INSERT）

## 3.5 コロンコマンド（COLON）
- `:w`  : 保存
- `:q`  : 変更がなければ終了（変更がある場合は拒否し、メッセージを表示）
- `:q!` : 変更を破棄して終了
- `:wq` : 保存して終了

未対応コマンドは `unknown command` として扱う。

## 3.6 検索（NORMAL/SEARCH）
- `/pattern` : 前方検索（リテラル一致、大小区別あり）
  - 検索開始位置は「現在カーソル位置の次の文字」から
  - 行をまたいでバッファ終端まで検索する（ラップアラウンドはしない）
  - 見つかった場合は一致箇所へカーソルを移動する
  - 見つからない場合はカーソル位置を維持し、メッセージに `Pattern not found` を表示する
- `n` : 直前の検索を前方に繰り返す

検索文字列の扱い:
- 正規表現は未対応（単純な部分文字列検索のみ）
- UTF-8 は「バイト列として一致」を基本とする（表示幅計算とは独立）
- カーソル位置が UTF-8 を壊さないよう、一致開始位置は UTF-8 文字境界に限る

# 4. 文字と表示幅（「それなりに正確」）

## 4.1 UTF-8 編集の成立条件
- カーソル位置（バイトインデックス）は UTF-8 文字境界に揃える
- INSERT では、入力バイト列を「先頭バイトから推定した必要長」まで組み立ててから挿入する
  - 厳密な UTF-8 検証（不正シーケンスの拒否）は行わない

## 4.2 表示幅ルール（初期版）
- ASCII 文字: 幅 1
- 非 ASCII（先頭バイトが 0x80 以上）: 幅 2 とみなす（簡易）
- タブ `\t`: タブストップ（既定 4）で次の桁に揃える

このルールは「全角が混ざるとカーソルが極端にズレる」問題を避けるための、実用寄りの近似である。

---

# 5. ファイル入出力

- 読み込み:
  - `\n` / `\r` は行末から除去して内部表現（行データ）に保持しない
  - ファイルが無い場合は空バッファ扱い
- 保存:
  - 各行の末尾に必ず `\n` を付けて書き出す
  - そのため、最終行も常に改行で終わる

---

# 6. 制限事項

- 1 行の最大バイト数に上限がある（既定 4096）
- 端末サイズ取得に失敗した場合は 24x80 とみなす
- 全画面再描画（差分描画なし）
- `--batch` は「擬似キー入力」専用で、表示は更新しない（編集結果の生成が目的）

---

# 7. 受け入れ条件（スモーク）

以下が自動スモークで確認できること。
- INSERT で ASCII と日本語（UTF-8）を入力し、`:wq` で保存できる
- NORMAL で `dd/yy/p` が動作し、`:wq` で保存できる
- NORMAL で `3dd/3yy` が動作し、`p` が複数行貼り付けできる
- NORMAL で `0/$` により行頭/行末へ移動できる
- NORMAL で `/pattern` と `n` により次の一致へ移動できる（日本語パターンを含む。見つからない場合はメッセージが出る）

スモークは [uim/tests/smoke_uim.sh](../uim/tests/smoke_uim.sh) を参照。
