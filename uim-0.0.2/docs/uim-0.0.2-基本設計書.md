# 1. 目的と非目的

## 1.1 目的
- UmuOS 上で研究用途の編集ができる「最小 vim ライク」エディタを提供する
- 日本語（UTF-8）を壊さない編集と、全角幅が混ざっても破綻しにくいカーソル表示を実現する
- 自動スモーク（`--batch`）で最低限の編集操作を再現可能にする

## 1.2 非目的
- vi/vim 互換の追求、差分描画、undo、置換などの高度機能
- 厳密な Unicode 幅算出
- 正規表現検索、インクリメンタル検索、検索ハイライト、後方検索

# 2. 全体構成

## 2.1 ディレクトリ構成
参照実装は `uim-0.0.2/uim` に置く。

- `include/` : 型・定数・API 宣言
- `src/` : 実装
- `tests/` : スモークテスト

## 2.2 モジュール分割（責務）

- `term`（端末 I/O）
  - raw mode（termios）
  - 1 キー読み取り（ESC シーケンスの最小解釈）
  - 画面制御（clear/move cursor/hide cursor/flush）
  - 端末サイズ取得

- `utf8`（UTF-8 境界/表示幅）
  - 文字境界の前後移動（prev/next）
  - 表示幅の近似（ASCII=1、非ASCII=2）
  - 表示列↔バイト位置変換（横スクロールとカーソル維持に使用）

- `buf`（行配列バッファ）
  - ファイル load/save
  - 行の削除/挿入（`dd`/`p`）
  - 行内のバイト挿入（INSERT）
  - UTF-8 1 文字削除（Backspace）
  - 行分割（Enter）

- `render`（描画）
  - 全画面再描画
  - スクロール（縦/横）
  - ステータス行、メッセージ行
  - カーソル位置の表示列計算（UTF-8 幅近似を使用）

- `editor`（編集・状態遷移）
  - モード管理（NORMAL/INSERT/COLON/SEARCH）
  - キー処理とコマンド実装（`dd/yy/p/a`, `/pattern`, `n`, `:w/:q/:wq/:q!`）
  - INSERT における UTF-8 組み立て
  - `--batch`（stdin 疑似入力）処理

- `main`（起動）
  - 引数解析（`--batch` / `--version`）
  - バッファロード
  - interactive/batch 実行

# 3. データ設計

## 3.1 エディタ状態（`uim_t`）
- `mode`: 現在モード
- `path`: 編集対象ファイルパス（保存先）
- `buf`: 行配列バッファ
- `cur_row` / `cur_col_byte`: カーソル位置（列はバイトインデックスで UTF-8 境界）
- `row_off` / `col_off`: スクロール（行オフセット、表示列オフセット）
- `yank`: ヤンクバッファ（1 行以上。`yy`/`Nyy`/`dd`/`Ndd` で格納し、`p` で複数行貼り付け可能）
- `colon[]`: コロンコマンド入力
- `search[]`: `/` 検索入力（SEARCH モード）
- `last_search[]`: 直前の検索パターン（`n` 繰り返し用）
- `status[]`: ステータスメッセージ（TTL で自然消滅）
- `pending`: NORMAL の 2 打鍵コマンドの 1 打鍵目（`d` / `y` / `g`）
- `num_prefix`: NORMAL の数値プレフィクス（例: `3dd`, `10G`）
- `u8buf/u8len/u8need`: INSERT の UTF-8 組み立て
- `running`: メインループ継続フラグ

## 3.2 バッファ（`uim_buf_t` / `uim_line_t`）
- 行配列として保持
- 各行は `\0` 終端の文字列（内部表現に改行は持たない）
- 編集操作により `modified` を 1 にする

# 4. 画面・操作設計

## 4.1 再描画
- interactive は入力処理のあとに `render` を呼び、可視領域全体を「上書き」で再描画する（差分描画はしない）
- 画面全消去（clear）は interactive への入場時/終了時に行い、毎フレームの clear は避ける（チラつき低減）
- カーソルは非表示→描画→表示に戻す

## 4.2 スクロール
- 縦スクロール: カーソル行が可視範囲に入るよう `row_off` を調整
- 横スクロール: カーソル表示列が可視範囲に入るよう `col_off` を調整
- 横スクロールの切り出しは「表示列→バイト位置」変換を用いる

## 4.3 ステータス/メッセージ
- 最下 2 行を使用
  - ステータス行: 反転表示（モード、ファイル、変更有無、行/列）
  - メッセージ行: COLON 中は `:...` を表示、通常は status を表示

---

# 5. 入力設計

## 5.1 interactive
- raw mode で 1 キーずつ受け取る
- ESC シーケンスは最小限（矢印キー、Home/End）を認識
- その他は 1 バイト入力として扱う（UTF-8 は複数バイトとして連続して届く想定）
- 端末に入力が溜まっている場合は、可能な範囲でまとめて処理してから 1 回だけ描画する（大量ペースト時の体感改善）

## 5.2 batch
- stdin からバイト列を読み、擬似キーとして editor に渡す
- `\x1b` は ESC、`\n/\r` は Enter、`\x7f/\b` は Backspace として解釈
- 描画は行わない（編集結果ファイルを得ることが目的）

---

# 6. エラー方針

- `load/save` 失敗は呼び出し側にエラーを返す
- `:w` / `:wq` の失敗はエディタを継続し、ステータスに `write failed` を表示する
- `:q` は modified の場合は拒否し、ステータスにメッセージを表示する

---

# 7. テスト方針

- `tests/smoke_uim.sh` により以下を自動確認する
  - INSERT で日本語を含む入力→ `:wq` 保存
  - `dd/yy/p` → `:wq` 保存
  - `/pattern` と `n` による前方検索の反復（日本語パターンを含む）
- テストは「バイナリが無ければビルド」する（musl-gcc 静的リンク）

---

# 8. ビルド方針

- `musl-gcc -static` を基本とする
- `-D_POSIX_C_SOURCE=200809L -D_XOPEN_SOURCE=700` を指定し、termios 等の宣言が見えることを前提にする
