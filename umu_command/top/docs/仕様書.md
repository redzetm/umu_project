# top（UmuOS用 ミニ top）仕様書

## 目的

- BusyBox 以外の最小実装で、UmuOS（ttyS/pts）上でも使いやすい `top` 相当を提供する。
- stable/dev どちらでも、`/umu_bin/top` を置くだけで利用可能にする（OS側の設定変更を前提にしない）。

## コマンド名

- バイナリ名: `top`

## オプション

- `-u USER`
  - 指定したユーザーのプロセスだけを表示する。
  - `USER` にはユーザー名または数値 UID を指定できる。
- `-h`
  - 使用方法を表示して終了する。

## 入出力

- 標準出力に画面を描画（ANSI制御：カーソル移動/消去）
- キー入力は可能なら `/dev/tty` から読む

## 更新間隔

- 既定: 2秒

## UI/操作

- `q` または `Q`: 終了
- その他のキーは無視（最小）

## 表示（top風）

### ヘッダ（概ね top と同型）

- 1行目: `top - <時刻> up <uptime>,  <users> users,  load average: <1> <5> <15>`
- 2行目: `Tasks: <total> total, <running> running, <sleeping> sleeping, <stopped> stopped, <zombie> zombie`
- 3行目: `%Cpu(s): <us> us, <sy> sy, <ni> ni, <id> id, <wa> wa, <hi> hi, <si> si, <st> st`
  - ここでは `/proc/stat` の差分から `us/sy/ni/id` を算出し、その他は 0.0 としてよい（最小）
- 4行目: `MiB Mem : <total> total, <free> free, <used> used, <buff/cache> buff/cache`
- 5行目: `MiB Swap: <total> total, <free> free, <used> used. <avail> avail Mem`

### プロセス一覧

- デフォルトはCPU使用率（直近サンプル間の差分）降順で上位N件を表示
- `-u USER` 指定時は、そのユーザーに属するプロセスだけを対象に並べ替えて表示する
- 表示カラム（top風・近似でOK）:

`PID USER PR NI VIRT RES SHR S %CPU %MEM TIME+ COMMAND`

- USER は `/proc/<pid>/status` の uid から `/etc/passwd` を引いて推定（無ければ数値）
- VIRT は `/proc/<pid>/stat` の `vsize`
- RES は `rss * page_size`
- SHR は `/proc/<pid>/statm` の shared から推定
- TIME+ は `(utime+stime)` を `CLK_TCK` で秒にして `MM:SS` 表示（近似）

## データソース

- `/proc/loadavg`
- `/proc/uptime`
- `/proc/stat`
- `/proc/meminfo`
- `/proc/<pid>/stat`, `/proc/<pid>/statm`, `/proc/<pid>/status`

## ビルド方針

- 依存は libc のみ（ncursesなし）
- UmuOS持ち込み用に静的リンク版を推奨
  - `make STATIC=1 CC=musl-gcc`

## 非目標

- top の全キー操作（P/M/Tなど）
- 折り返し幅の厳密計算、色付け、検索
- 複数ユーザー同時指定（例: `-u root,tama`）
