# uls（UmuOS用 ls コマンド）仕様書

## 目的

- BusyBox の `ls` が日本語ファイル名を `???` のように表示してしまう環境でも、UmuOS 上でファイル名をそのまま表示できる `ls` 互換コマンドを提供する。
- UmuOS の stable/dev 系を大きく改変せずに、`/umu_bin/ls` を配置して差し替えできるようにする。

## コマンド名

- バイナリ名: `ls`
- 以降、この実装を `uls` と呼ぶ（ソース/仕様上の呼称）。

## 設計方針

- 最小実装を優先する。
- ファイル名表示時に独自エスケープや文字置換を行わず、ディレクトリエントリで得た名前をそのまま標準出力へ出す。
- `ll` の代替に使いやすいよう、`-l` を最優先で実装する。

## 対応オプション

- `-l`
  - 長形式表示
- `-a`
  - `.` で始まる名前も表示
- `-1`
  - 1 行 1 件表示

## 非対応オプション

- `-b`, `--color`, `-h`, `-R` などの拡張機能は持たない。
- 未対応オプション指定時はエラー終了する。

## 引数

- 引数なし: カレントディレクトリを表示
- 引数あり: 指定したファイルまたはディレクトリを表示
- 複数引数: 指定順に表示する

## 出力

- 通常表示: 1 行 1 件で名前をそのまま出す
- `-l` 指定時: 以下を表示する
  - 種別とパーミッション
  - リンク数
  - ユーザー名（取得できなければ UID 数値）
  - グループ名（取得できなければ GID 数値）
  - サイズ
  - 更新時刻
  - 名前
- 標準出力が TTY のときは基本色付けを行う
  - ディレクトリ: 青
  - シンボリックリンク: シアン
  - 実行可能ファイル: 黄
  - `NO_COLOR` が設定されている場合や `TERM=dumb` の場合は色を出さない

## 日本語ファイル名に対する方針

- UTF-8 を特別扱いせず、「バイト列をそのまま表示する」方針とする。
- そのため、端末側が UTF-8 を表示できるなら日本語名もそのまま見える。
- BusyBox `ls` のような独自置換で `?` に変える挙動は行わない。
- 色付けしても名前本体のバイト列は変更しない。ANSI エスケープを前後に付けるだけとする。

## 終了ステータス

- 0: 正常終了
- 1: エラー（`stat` 失敗、`opendir` 失敗、未対応オプション等）

## ビルド方針

- ホスト（開発機）でビルド可能
- UmuOS へ持ち込む場合は静的リンク版を推奨
  - `make STATIC=1 CC=musl-gcc`

## UmuOS 持ち込み手順（推奨）

- UmuOS へ持ち込むバイナリは、動的リンク版ではなく静的リンク版を使う。
- ホスト側で次を実行する。

```sh
make clean
make STATIC=1 CC=musl-gcc
file ls
```

- `file ls` の結果に `statically linked` が含まれることを確認する。
- その後、UmuOS 側へ配置する。

```sh
cp -f ls /umu_bin/ls
chown root:root /umu_bin/ls
chmod 0755 /umu_bin/ls
ls -l /umu_bin/ls
```

- `ll` を `ls -l` の alias にしている場合は、そのまま長形式表示の代替として使える。

## 非目標

- GNU coreutils `ls` の完全互換
- 色付け、列表示、自動幅調整
- 再帰表示
- 非表示文字のエスケープ表示
