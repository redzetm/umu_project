# umore（UmuOS用 more コマンド）仕様書

## 目的

- BusyBox の `more` が環境（CR/NL等）によって期待通りに動かない場合でも、UmuOS上で確実に「1行送り / 1ページ送り」ができる pager を提供する。
- UmuOS の stable（例: 0.1.7-base-stable）を改変せずに、`/umu_bin/more` を置くことで BusyBox `more` を置き換えられるようにする。

## コマンド名

- バイナリ名: `more`
- 以降、この実装を `umore` と呼ぶ（ソース/仕様上の呼称）。

## 入出力

- 入力
  - 引数にファイルが指定された場合: そのファイルを表示
  - 引数が無い場合: 標準入力を表示（例: `uman uman | more`）
- 出力
  - 標準出力へ表示
- 操作入力
  - 標準入力がパイプ等で塞がることがあるため、操作入力は可能なら `/dev/tty` から読む

## 端末互換（重要）

- 「Enterキー入力」は `\n` だけでなく `\r` も Enter として扱う（CR/NL差異吸収）。
- 端末行数の取得は次の優先順位で決める:
  1. `ioctl(TIOCGWINSZ)` で rows
  2. 環境変数 `LINES`
  3. 既定値 24

## 画面制御

- 画面消去などの高度な制御は行わない（最小）。
- ページングは「表示した行数」で行う（折り返し幅は考慮しない）。

## 画面下プロンプト

- ページ末尾で操作待ちに入る。
- 表示文言（固定）:

  `(Enter:next line Space:next page Q:quit R:show the rest)`

## キー操作

- `Space` : 次ページ
- `Enter` : 次行（`\r` または `\n`）
- `q`/`Q` : 終了
- `r`/`R` : 残りを全表示して終了（ページング無効化）

## 終了ステータス

- 0: 正常終了
- 1: エラー（ファイルを開けない等）

## オプション

- 最小実装のため、オプションは持たない（将来拡張余地）。

## ビルド方針

- ホスト（開発機）でビルド可能
- UmuOSへ持ち込む想定では、静的リンク版を推奨
  - `make STATIC=1 CC=musl-gcc`

## UmuOS 持ち込み手順（推奨）

- UmuOS へ持ち込むバイナリは、動的リンク版ではなく静的リンク版を使う。
- ホスト側で次を実行する。

```sh
make clean
make STATIC=1 CC=musl-gcc
file more
```

- `file more` の結果に `statically linked` が含まれることを確認する。
- その後、UmuOS 側へ配置する。

```sh
cp -f more /umu_bin/more
chown root:root /umu_bin/more
chmod 0755 /umu_bin/more
ls -l /umu_bin/more
```

- 動的リンク版を持ち込むと、UmuOS 側で `not found` になることがある。

## 非目標

- `less` 相当の検索・ジャンプ機能
- 行の折り返し幅計算、マルチファイル連結表示など
