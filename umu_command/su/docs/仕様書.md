# su（UmuOS用 自作 su）仕様書

## 目的

- UmuOS 上で「setuid による euid 変化（権限昇格の成立条件）」を観測できる、最小の `su` を提供する。
- OS 本体（initramfs / rootfs）側の実装から、コマンド実装（ユーザーランドツール）を切り出す。
- BusyBox ash（`/bin/sh`）でも、任意で持ち込む ush でも、対話で固まらずに実行できることを前提にする。

## コマンド名

- バイナリ名: `su`
- 想定配置: `/umu_bin/su`

## 前提（成立条件）

- `su` バイナリは **root 所有**で **setuid bit** が立っていること。
  - 例: `chown root:root /umu_bin/su` / `chmod 4755 /umu_bin/su`
- `/etc/shadow` が存在し、`root:` のエントリがあること。
- `/etc/shadow` の root パスワードがロック（`!`/`*`）や空でないこと。
- `nosuid` マウントの場合は成立しない（観測点）。

## 動作仕様

- 実行時、`/etc/shadow` から `root` のパスワードハッシュを読み取る。
- `Password: ` を表示し、パスワードを入力させる。
  - パスワード入力は可能なら `/dev/tty` から読み、標準入力がパイプ等でも対話できるようにする。
  - 入力中は echo を無効化する。
- `crypt(3)` で入力パスワードを検証し、一致したら `setgid(0)` → `setuid(0)` の後に `/bin/sh` を起動する。

## 入出力

- 入力: `/dev/tty`（可能なら）
- 出力: `stderr`（プロンプト/エラー）

## 終了ステータス

- 0: 認証成功し、`/bin/sh` の起動に移行（通常は `execl` するため戻らない）
- 1: エラー（euid!=0、shadow 読み出し失敗、認証失敗、setuid/setgid 失敗など）

## ビルド方針

- `crypt()` のリンク条件は環境差がある（追加ライブラリ不要 / `libcrypt` / `libxcrypt`）。
  - そのためビルドは `（追加無し）→ -lcrypt → -lxcrypt` の順で試行する。

- UmuOS 持ち込み用に静的リンク版を推奨:
  - `make STATIC=1 CC=musl-gcc`

## インストール例（概念）

- UmuOS の rootfs 側へ配置して setuid を付ける:

```sh
# 例: ホスト側で disk.img を loop mount 済みの前提
sudo cp -f ./su /mnt/umuos016/umu_bin/su
sudo chown root:root /mnt/umuos016/umu_bin/su
sudo chmod 4755 /mnt/umuos016/umu_bin/su
```

## 非目標

- ユーザー指定（`su tama` 等）
- `-c` 等のオプション
- PAM / 監査ログ / 環境変数整備などの本格実装
