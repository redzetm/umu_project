# ush-0.0.5-仕様書.md
UmuOS User Shell (ush) — 仕様書（0.0.5 / 拡張）  
Target OS: UmuOS-0.1.7-base-stable（想定）

この文書は、ush 0.0.4 の仕様を引き継ぎつつ、ush 0.0.5 で「普段使いで困る」ことが多い以下を追加する。

- コマンド置換 `$(...)`
- 位置パラメータ `$1` 等（スクリプト引数）

0.0.5 の設計方針は 0.0.4 と同様。

注意（このリポジトリ運用）:
- 実装の唯一の正は [ush-0.0.5-詳細設計書.md](ush-0.0.5-詳細設計書.md) とする。
- 本書は仕様（要約）であり、矛盾がある場合は詳細設計書を優先する。

- 目標
  - ash に逃げずに「日常操作で詰まらない」ラインを少し上げる
  - POSIX 準拠は目的にしない（ただし挙動は曖昧にせず仕様で固定）
- 設計思想
  - 最小限・高速・シンプル
  - 未対応は「検出してエラー」または「決め打ちで単純化」
  - 誤動作より観測可能性（何が起きたかが読める）を優先

# 1. 目的と非目的

## 1.1 目的
- UmuOS におけるユーザー向け対話シェルとして、軽量で観測可能な操作環境を提供する
- 直列処理・成功/失敗分岐（`&&` / `||`）中心の「小物スクリプト」を ush で書けるようにする
- 0.0.4 までの仕様に加えて、0.0.5 では以下を ush で扱えるようにする
  - コマンド置換 `$(...)`
  - 位置パラメータ（`$0` `$1`..`$9` `$#`）
  - スクリプトモードでの引数受け渡し（`ush script.ush a b c`）

## 1.2 非目的（0.0.5ではやらない）
- POSIX sh / bash 互換の追求
- ジョブ制御（バックグラウンド実行 `&`、fg/bg、`jobs` 等）
- 制御構文（`if/for/while/case` 等）による本格スクリプト
- 位置パラメータの完全互換
  - `$@` / `$*` の「複数語展開」
  - `shift` / `set` / `getopts` / 関数・スコープ
- コマンド置換の完全互換
  - バッククォート `` `...` ``
  - 入れ子 `$( $(...) )`
  - 高度なクォート相互作用（POSIX互換のフィールド分割/クォート保持など）
- ヒアドキュメント（`<<` / `<<<`）
- 高度な FD 操作（例: `2>file`、`n>file`、`2>&1` 等）
- 多段パイプ（`a|b|c`）

本格スクリプトが必要な場合は `/bin/sh`（BusyBox ash）で実行する。

# 2. 位置づけ（ash と ush）

## 2.1 役割分担
- `/umu_bin/ush` : ユーザー向け対話操作・軽量スクリプト（本仕様）
- `/bin/sh` : 互換性重視のスクリプト実行（BusyBox ash）

## 2.2 運用方針（推奨）
- 対話操作は ush を基本にする
- スクリプトは以下で使い分ける
  - ush: 直列処理・簡易な分岐（`&&`/`||`）・コマンド区切り `;`・（必要なら）グロブ程度までで足りるもの
  - sh: 分岐/ループ/高度なリダイレクト/多段パイプ/互換性が必要なもの

# 4. 機能一覧（0.0.5）

## 4.1 対話シェル（0.0.4 継承）
- プロンプト表示
- 1 行入力（EOF/Ctrl-D で終了）
- 行編集（最低限）
  - Backspace/Delete/左右矢印/上下（履歴）
  - Tab 補完（0.0.4 拡張）
- PATH 検索（未設定時は `/umu_bin:/sbin:/bin`）
- 外部コマンド実行（fork/exec）
- builtins: `cd`, `pwd`, `export`, `exit`, `help`
- SIGINT の扱い（親は落ちない、子はデフォルト）

## 4.2 シェル言語（小物スクリプト向け）
### 0.0.4 から継承するもの
- クォート
  - シングルクォート `'...'`（展開なし）
  - ダブルクォート `"..."`（変数展開とコマンド置換のみ）
- 展開
  - `$VAR`（環境変数）
  - `${VAR}`（環境変数）
  - `$?`（直前の終了コード）
  - `~` / `~/...`（チルダ展開）
- 演算子
  - `|`（1段パイプのみ）
  - `&&` / `||`（短絡評価）
  - `;`（逐次実行）
  - `<` `>` `>>`（リダイレクト）
- コメント
  - `#`（行中コメント対応、ただしトークン先頭のみ）
- 最小のバックスラッシュエスケープ（10章）
- グロブ（`* ? [ ]`）の展開（12章）

### 0.0.5 で追加するもの
- 位置パラメータ（11.3章）
  - `$0` `$1`..`$9` `$#`
- コマンド置換 `$(...)`（11.4章）

# 5. 実行モデル

## 5.1 `last_status`（0.0.4 継承）
- ush は `last_status`（初期値 0）を保持する
- 外部コマンド実行後
  - 通常終了: `WEXITSTATUS(status)`
  - シグナル終了: `128 + WTERMSIG(status)`
- builtin 実行後
  - 成功: 0
  - 実行エラー: 1
  - 構文/引数不正: 2
- 対話モードの EOF（Ctrl-D）は `exit(last_status)`

外部コマンド探索の慣習：
- 未発見: 127
- 実行不可（権限等）: 126

## 5.2 エラーメッセージ（0.0.4 継承）
- stderr に出力し、先頭に `ush:` を付与する
- 代表例
  - `ush: unsupported syntax`
  - `ush: syntax error`
  - `ush: open: <strerror>`

## 5.3 失敗時の優先順位（0.0.4 継承）
同一行の評価で複数の「失敗要因」があり得るため、0.0.5 でも次の順序で `last_status` を決める。

1) 字句/構文段階の失敗
  - 未対応構文検出: `last_status = 2`（その行は実行しない）
  - 構文エラー: `last_status = 2`（その行は実行しない）
2) 展開段階の失敗
  - 変数展開/コマンド置換で `syntax error` 相当: `last_status = 2`（その行は実行しない）
3) リダイレクトの失敗
  - `open()` 等に失敗: `last_status = 1`（その行は実行しない）
4) 実行（fork/exec）段階の失敗
  - コマンド探索に失敗: 126/127
5) 実行結果
  - 外部コマンド: 0..255 または `128 + signal`
  - builtins: 0/1/2

# 6. トークナイズ（字句）

## 6.1 区切り（0.0.4 継承）
- 空白（スペース/タブ）で区切る
- 連続空白は 1 つの区切りとして扱う（先頭/末尾は無視）

## 6.2 コメント（`#`）（0.0.4 継承）
未クォートで、かつ「トークン先頭の `#`」をコメント開始として扱う。

- `# ...` / `   # ...` はその行を無視
- `echo hi # comment` の `#` 以降を無視
- `echo foo#bar` の `#` はトークン途中なのでコメント扱いしない

## 6.3 演算子（独立トークン）（0.0.4 継承）
空白の有無に関わらず、以下を独立トークンとして認識する。

- `|` `&&` `||` `<` `>` `>>` `;`

ただし、クォート内では演算子は文字として扱う。

## 6.4 クォート（0.0.4 継承）
### 6.4.1 シングルクォート（`'...'`）
- 囲まれた範囲は 1 トークン
- 展開なし
- 閉じクォートがない場合はエラー

### 6.4.2 ダブルクォート（`"..."`）
- 囲まれた範囲は 1 トークン
- ダブルクォート内は空白や演算子も文字
- ダブルクォート内で行う展開は **変数展開**（`$VAR`, `$?`, `${VAR}`, 位置パラメータ）と **コマンド置換**（`$(...)`）のみ
- 閉じクォートがない場合はエラー

## 6.5 バックスラッシュ（0.0.4 継承）
- バックスラッシュによる **行継続（`\` + 改行）** は未対応
- ただし、最小の「1文字エスケープ」を導入する（詳細は 10 章）

## 6.6 未対応トークンの検出（0.0.4から一部緩和）
- 単体 `&` は未対応（`&&` のみ対応）
- 未クォートで「単体トークン」として現れる `{` `}` は未対応
  - ただし WORD 内の `{` `}` は `${NAME}` のために許可する（6.4章のとおり、クォート内も文字として扱う）

0.0.5 では `$(...)` を導入するため、未クォートにおける `(` `)` は以下の扱いとする。

- `$(...)` を構成する `(` `)` は許可
- 上記以外の `(` `)` は未対応として検出（`unsupported syntax`）

補足:
- クォート内（`'...'` / `"..."`）では `(` `)` は文字として扱い、未対応検出はしない。
- クォート内でも `$(...)` はコマンド置換として展開対象になる（11.4章）。

未対応検出時:
- stderr: `ush: unsupported syntax`
- `last_status=2`

# 7. 構文（パーサ）（0.0.4 継承）

## 7.1 優先順位
- `|` が高い
- `&&` / `||` が中
- `;` が低い（左から順に実行）

## 7.2 文法（概略）
コメントはトークナイザ段階で除去されている前提とする。

```
line        := seq
seq         := list (";" list)*
list        := pipeline ( ("&&" | "||") pipeline )*
pipeline    := command ("|" command)?
command     := words redirects?
words       := WORD+
redirects   := ("<" WORD)? ((">"|">>") WORD)?
```

### 7.2.1 リダイレクトの位置（0.0.4 継承）
- リダイレクトは **コマンド引数（words）の後ろにまとめて書く**
  - OK: `cat a.txt > out.txt`
  - NG: `cat > out.txt a.txt`（`syntax error`）
- 同一コマンド内で許可するリダイレクト
  - `<` は 0 個または 1 個
  - `>` / `>>` は 0 個または 1 個

# 8. 実行（exec）（0.0.4 継承）

## 8.1 条件実行（`&&`, `||`）
- `A && B`
  - A が成功（終了コード 0）なら B を実行
  - A が失敗（非 0）なら B は実行しない
- `A || B`
  - A が成功なら B は実行しない
  - A が失敗なら B を実行

## 8.2 `;`（逐次実行）（0.0.4 継承）
- `A ; B` は `A` を実行してから必ず `B` を実行する
- `last_status` は `B` の結果で更新される

## 8.3 リダイレクト失敗時の扱い（0.0.4 継承）
- リダイレクト対象ファイルは **fork 前に親で `open()` して成功確認**する
- 1 つでも失敗したら、その行は **1 つも fork/exec しない**（部分実行しない）

## 8.4 `ENOEXEC` のみ `/bin/sh` フォールバック（0.0.4 継承）
- `execve()` が `ENOEXEC` のときのみ `/bin/sh` へフォールバックする

# 11. 展開（変数/位置パラメータ/コマンド置換）（0.0.5 拡張）

## 11.1 変数展開（環境変数）
- 対応
  - `$NAME`
  - `${NAME}`
  - `$?`
- 未定義変数は空文字
- シングルクォート内は展開しない

未対応（検出してエラー）
- `${NAME:-default}` 等の派生形

## 11.2 `$?`（直前の終了コード）
- 直前の評価での `last_status` を 10 進表記で展開する

## 11.3 位置パラメータ（0.0.5 追加）
0.0.5 は「スクリプトへ引数を渡して `$1` 等で参照したい」を最小仕様で満たす。

- 対応
  - `$0`
  - `$1`..`$9`
  - `$#`

### 11.3.1 値の定義
- 対話モード
  - `$0` は `ush`
  - `$1`..`$9` は空文字
  - `$#` は `0`
- スクリプトモード（`ush script.ush arg1 arg2 ...`）
  - `$0` は `script.ush`（`ush` に渡したパス文字列）
  - `$1` は `arg1`
  - `$2` は `arg2`
  - ...
  - `$#` は `argN` の個数（`arg1..argN` の数）

### 11.3.2 仕様上の制限
- `$10` 以降は未対応（検出して `unsupported syntax`）
- `$@` / `$*` は未対応（検出して `unsupported syntax`）

## 11.4 コマンド置換 `$(...)`（0.0.5 追加）

### 11.4.1 基本動作
- `$(cmdline)` は、`cmdline` を ush の「1行」として評価し、その標準出力を文字列として埋め込む。
- `cmdline` は ush の構文（7章）を用いる（`|`/`&&`/`||`/`;`/リダイレクト含む）。

### 11.4.2 出力文字列の正規化
コマンド置換の出力は、誤動作より単純さを優先して次の正規化を行う。

- `\r` と `\n` はスペース `' '` に置換する
- 末尾の連続空白は削除してよい

### 11.4.3 `last_status` との関係
- コマンド置換の実行結果（終了コード）は、**その時点の `last_status` を上書きしない**。
  - 目的: 1行の評価中に `last_status` の意味が揺れないようにする。
- ただし、コマンド置換の解析が失敗（構文エラー/未対応検出/長すぎ）した場合は、その行は実行せず `last_status=2` とする。

### 11.4.4 制限（0.0.5 の決め打ち）
- 入れ子 `$( $(...) )` は未対応（検出して `unsupported syntax`）
- バッククォート `` `...` `` は未対応（検出して `unsupported syntax`）
- 置換結果のフィールド分割（空白で argv を増やす等）は行わない
  - 置換結果は「1つのWORD」に埋め込まれる

# 16. スクリプトモード（0.0.5 拡張）

## 16.1 起動形式
- 対話: `ush`

- `exit` builtin が実行されたらその時点で終了
- スクリプト引数は位置パラメータ（11.3章）として参照できる

# 21. 受け入れ（仕様観点）

## 21.1 位置パラメータ
- スクリプトモードで `$1`..`$9` と `$#` が参照できる
- 対話モードで `$1`..`$9` が空になる（エラーではない）
- `$10`、`$@`、`$*` は `unsupported syntax` になる

## 21.2 コマンド置換
- `echo $(echo hi)` が `hi` を出力する
- `echo X$(echo hi)Y` が `XhiY` を出力する
- `$(...)` の中で `;` `&&` `||` `|` が使える
- `$(...)` の中で構文エラー/未対応構文を起こした場合、その行全体が `syntax error` / `unsupported syntax` 扱いで止まる
- `` `...` `` は `unsupported syntax` になる

- 多段パイプ
- ヒアドキュメント
- 高度なFD操作
- 位置パラメータ完全互換（`$@` / `shift` 等）
- コマンド置換完全互換（バッククォート、入れ子、フィールド分割）
