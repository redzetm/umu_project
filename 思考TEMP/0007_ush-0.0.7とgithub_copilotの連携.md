# 0007 ush-0.0.7 と GitHub Copilot の連携（C案固定メモ）

目的: UmuOS（最小ユーザーランド）上の ush から、GitHub Copilot 相当の支援を「日常的に」使えるようにする。

結論: **C案（ホストプロキシ / ゲスト送受信のみ）** を採用する。
UmuOS(192.168.0.202) は BusyBox の `nc` / `telnet` でプロキシへ質問を投げ、Copilot 実行（`gh copilot`）・認証・外部ネットアクセスは 192.168.0.203（Ubuntu等）へ寄せる。

※ここでの「ゲスト送受信のみ」は、**ゲスト側は入力を送って結果を表示するだけ**で、Copilotの実行・認証・依存管理はホスト側に集約する、という意味。

---

## 前提・制約

- UmuOS-0.1.7-base-stable の中で完結したい（UmuOS側に重い依存を持ち込まない）
- バイナリ持ち込み運用
- disk.img（永続ディスク）への追加は問題ない
- UmuOS の BusyBox に `nc` と `telnet` がある（重要）

---

## C案とは（ここで言う「プロキシ」）

ここでの「プロキシ」は **HTTPプロキシ（forward proxy）ではなく**、Copilotの実行を“代行”する **アプリケーションプロキシ**。

- UmuOS(202):
	- 入力（質問）を 203 に送る
	- 応答（テキスト）を受け取って表示する
- Proxy(203):
	- 受け取った入力を `gh copilot` に渡して実行
	- 生成結果を UmuOS に返す

これにより、UmuOS側に次を持ち込む必要が薄くなる:

- OAuth認証（トークン保存）
- TLSルートCA / DNS / 時刻同期
- `gh` / `gh-copilot` の依存（node等が出た時のコスト）

---

## 方式（UmuOS→Proxy の通信）

UmuOS側が `nc` / `telnet` を使えるため、**TCP（行ベース）** を採用する。

- 接続先: `192.168.0.203:<PORT>`（例: `7777`）
- 1リクエスト = 1コネクション（実装を単純化）
- リクエストは1行（UTF-8想定）
- レスポンスは複数行 + 終端行

---

## 最小プロトコル（提案）

### リクエスト

形式:

`<verb> <topic> <text...>\n`

- `verb`: `suggest` / `explain`
- `topic`: `shell`（当面固定）
- `text`: 自然文の質問や、説明してほしいコマンド

例:

- `suggest shell netstatの使い方を教えてください\n`
- `explain shell netstat -tulpen\n`

### レスポンス

- 本文: 任意行数
- 終端: `\n.\n`（ドットだけの行）

---

## UmuOS 側のオペレーション（例）

### 1) 直接 `nc` で叩く（最小）

#### netstat の使い方を聞く

```sh
printf 'suggest shell %s\n' 'netstatの使い方を教えてください' | nc 192.168.0.203 7777
```

共有鍵（token）を使う場合（例）:

```sh
printf 'suggest shell %s %s\n' 'CHANGE_ME' 'netstatの使い方を教えてください' | nc 192.168.0.203 7777
```

#### コマンドの意味を説明させる

```sh
printf 'explain shell %s\n' 'netstat -tulpen' | nc 192.168.0.203 7777
```

メモ:

- Proxyがレスポンス後にコネクションを閉じる設計にする（`nc` が待ち続けるのを避ける）
- BusyBox `nc` が `-w <sec>`（タイムアウト）をサポートしているなら、必要に応じて付ける

### 2) `telnet` で叩く（デバッグ向け）

```sh
telnet 192.168.0.203 7777
```

接続後に1行投げる:

`suggest shell netstatの使い方を教えてください`

### 3) ush で短い“ラッパー”を用意する（任意）

運用としては、UmuOS 側に `/umu_bin/copilot` のような小さなスクリプト/バイナリを置いて、

- `copilot suggest 'netstatの使い方を教えてください'`

のように呼べると日常利用が楽。
ただし **最初の実装検証は 1) の `nc` 直叩きで十分**。

---

## Proxy(192.168.0.203) 側の役割（やること一覧）

最低限:

- TCPで待ち受け
- 1行リクエストを受け取る
- `gh copilot suggest` / `gh copilot explain` を実行
- 結果（stdout）を返し、最後に `.` 行で閉じる

推奨:

- 同時実行数制限（負荷対策）
- 1リクエストの最大サイズ制限（例: 4KB〜16KB）
- タイムアウト（例: 15〜30秒）
- ログ（入力/出力を全部残すかは要検討）

---

## Proxy 側のセットアップ（例）

### 1) `gh` と `gh-copilot` をホストに用意

- `gh auth login`（OAuth）でログイン
- `gh extension install github/gh-copilot`
- `gh copilot suggest -t shell 'echo hello'` がホストで動くことを確認

### 2) プロキシを起動

実装はなんでも良いが、重要なのは「入力をシェル解釈しない」こと。
（コマンド文字列連結ではなく、引数配列で `gh` を起動する）

## Proxy 最小実装（Python例）

前提:

- 192.168.0.203 側で `python3` が使える
- 203 側で `gh auth login` 済み
- 203 側で `gh extension install github/gh-copilot` 済み

起動例（203側）:

```sh
python3 copilot_proxy.py --bind 0.0.0.0 --port 7777 --allow 192.168.0.202
```

メモ: 可能なら `--bind 192.168.0.203` のように、待受をLAN内IPへ絞る。

共有鍵を使う場合:

```sh
python3 copilot_proxy.py --bind 0.0.0.0 --port 7777 --allow 192.168.0.202 --token 'CHANGE_ME'
```

実装例（ファイル名: `copilot_proxy.py`）:

```python
#!/usr/bin/env python3

import argparse
import socketserver
import subprocess
from typing import Optional


def run_gh_copilot(verb: str, text: str, timeout_sec: int) -> str:

    if verb == "suggest":
        cmd = ["gh", "copilot", "suggest", "-t", "shell", text]
    elif verb == "explain":
        cmd = ["gh", "copilot", "explain", text]
    else:
        return f"ERROR: unsupported verb: {verb}\n"

    try:
        proc = subprocess.run(
            cmd,
            text=True,
            capture_output=True,
            timeout=timeout_sec,
        )
    except FileNotFoundError:
        return "ERROR: gh not found on proxy host\n"
    except subprocess.TimeoutExpired:
        return "ERROR: gh copilot timed out\n"

    out = ""
    if proc.stdout:
        out += proc.stdout
        if not out.endswith("\n"):
            out += "\n"
    if proc.stderr:
        out += "[stderr]\n" + proc.stderr
        if not out.endswith("\n"):
            out += "\n"
    if not out:
        out = f"ERROR: empty response (exit={proc.returncode})\n"
    return out


class Handler(socketserver.StreamRequestHandler):
    server: "CopilotProxyServer"  # type: ignore[assignment]

    def handle(self) -> None:
        peer_ip = self.client_address[0]
        if self.server.allow_ip is not None and peer_ip != self.server.allow_ip:
            self.wfile.write(b"ERROR: forbidden\n.\n")
            return

        raw = self.rfile.readline(self.server.max_line_bytes + 1)
        if not raw:
            self.wfile.write(b"ERROR: empty request\n.\n")
            return
        if len(raw) > self.server.max_line_bytes:
            self.wfile.write(b"ERROR: request too large\n.\n")
            return

        try:
            line = raw.decode("utf-8", errors="replace").rstrip("\r\n")
        except Exception:
            self.wfile.write(b"ERROR: decode failed\n.\n")
            return

        parts = line.split(" ", 2)
        if len(parts) < 3:
            self.wfile.write(b"ERROR: bad request format\n.\n")
            return

        verb, topic, rest = parts[0], parts[1], parts[2]
        if topic != "shell":
            self.wfile.write(b"ERROR: topic must be shell\n.\n")
            return

        # 共有鍵（任意）: `<verb> shell <token> <text...>`
        if self.server.token is not None:
            token_parts = rest.split(" ", 1)
            if len(token_parts) < 2:
                self.wfile.write(b"ERROR: token required\n.\n")
                return
            token, rest = token_parts[0], token_parts[1]
            if token != self.server.token:
                self.wfile.write(b"ERROR: invalid token\n.\n")
                return

        output = run_gh_copilot(verb=verb, text=rest, timeout_sec=self.server.gh_timeout_sec)
        self.wfile.write(output.encode("utf-8", errors="replace"))
        self.wfile.write(b".\n")


class CopilotProxyServer(socketserver.ThreadingTCPServer):

    allow_reuse_address = True

    def __init__(
        self,
        server_address,
        RequestHandlerClass,
        *,
        allow_ip: Optional[str],
        token: Optional[str],
        max_line_bytes: int,
        gh_timeout_sec: int,
    ):
        super().__init__(server_address, RequestHandlerClass)
        self.allow_ip = allow_ip
        self.token = token
        self.max_line_bytes = max_line_bytes
        self.gh_timeout_sec = gh_timeout_sec


def main() -> None:

    ap = argparse.ArgumentParser()
    ap.add_argument("--bind", default="0.0.0.0")
    ap.add_argument("--port", type=int, default=7777)
    ap.add_argument("--allow", default=None, help="Allow only this peer IP (e.g. 192.168.0.202)")
    ap.add_argument("--token", default=None, help="Optional shared token")
    ap.add_argument("--max-line-bytes", type=int, default=4096)
    ap.add_argument("--gh-timeout-sec", type=int, default=30)
    args = ap.parse_args()

    with CopilotProxyServer(
        (args.bind, args.port),
        Handler,
        allow_ip=args.allow,
        token=args.token,
        max_line_bytes=args.max_line_bytes,
        gh_timeout_sec=args.gh_timeout_sec,
    ) as srv:
        print(
            f"listening on {args.bind}:{args.port} allow={args.allow} "
            f"token={'yes' if args.token else 'no'}"
        )
        srv.serve_forever()


if __name__ == "__main__":

    main()
```

## セキュリティ設計（最低限）

LAN内でも以下はやる。

- 待受は「202からのみ許可」
	- 例: 203のFWで `192.168.0.202` のみ許可
- 共有鍵（簡易トークン）をリクエストに含める（必要なら）
	- 例: `suggest shell <token> <text...>`
- 入力をシェル解釈しない（注入対策）
- サイズ制限とタイムアウト（DoS対策）

---

## A案（ゲスト完結）を“後回し”にする理由

UmuOSで `gh copilot` を直接動かす場合、少なくとも以下の成立条件が必要になりがち。

- TLSルートCA
- DNS
- 時刻同期（TLSで時刻ずれは致命的）
- OAuth認証とトークン保存
- `gh-copilot` の実体依存（node等が出る可能性）

このコストを避けつつ日常利用へ持っていくため、0.0.7ではC案を主案にする。
