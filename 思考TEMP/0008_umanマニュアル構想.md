# 0008 uman マニュアル構想（思考メモ）

目的: UmuOS（最小ユーザーランド）で「日本語で読めるコマンド説明」を提供する。
- roff / groff / man-db の整備は前提にしない
- UmuOSに重い依存を増やさない
- 将来 GitHub / Web に転載しやすい形式で管理する

結論（方針）: **ページは Markdown（.md）で管理し、UmuOS側は `uman <name>` で該当ページをそのまま標準出力へ流す**。
長いページは `uman netstat | more` のように既存のパイプで閲覧する。

---

## 想定ユーザー体験（UX）

- `uman netstat` → 日本語Markdownがそのまま出力される
- `uman nc` → 同様
- `uman`（引数なし）→ 使い方を表示
- （任意）`uman -k <word>` → ページを簡易検索

非目標:
- 本物の `man(1)` 互換（roff解釈、セクション番号、manpath整備など）
- 端末装飾（太字/下線/色など）

---

## データ形式

### ページ形式
- UTF-8 の Markdown（`.md`）
- 見出し/箇条書き/コードブロック中心で、**装飾なしの素表示でも読める**ように書く

### ページ命名
- コマンド名 = ファイル名（拡張子は `.md`）
	- 例: `netstat` → `netstat.md`
	- 例: `nc` → `nc.md`

---

## ディレクトリ（管理用 / 配置用）

### リポジトリ内（管理用）
案:
- `uman/pages/ja/<name>.md`

理由:
- GitHub上でそのまま読みやすい
- 将来 `uman/pages/en/` を増やす余地がある

### UmuOS（disk.img）内（配置用）
案:
- `/usr/share/uman/ja/<name>.md`

理由:
- 「データ置き場」として直感的
- `/umu_bin/`（実行系）と分離できる

---

## `uman` コマンド仕様（最小）

### 検索パス
- 既定: `/usr/share/uman/ja`
- （任意）環境変数で上書き可能: `UMAN_PATH`
	- 例: `UMAN_PATH=/run/uman/ja`（揮発運用用）

### 仕様（最小）
- `uman <name>`
	- `<name>` が英数/`_`/`-` の範囲なら許可（パストラバーサル対策）
	- `<path>/<name>.md` を開いて stdout へコピー
	- 見つからなければ、エラーメッセージを出して終了コード非0

### 仕様（任意・後回し）
- `uman -w <name>`: 実体ファイルのパスを表示
- `uman -k <word>`: ページ群を簡易grepして候補一覧を出す
- `uman --list`: ページ一覧

### ページャについて
- `uman` 自体はページャを起動しない
- 利用側が `| more` 等で対応する

---

## BusyBox `man` との関係

- BusyBox に `man` があっても、`uman` は別コマンドとして提供する（衝突回避）
- 「`man netstat` で呼びたい」場合は、後から以下のどちらかで対応できる
	- ush 側で `alias man=uman`
	- `/umu_bin/man` を `uman` への薄いラッパーにする（運用判断が固まってから）

---

## ページのテンプレ（推奨）

各ページは、最低限この構造にすると「探しやすい・差分が読みやすい」。

```markdown
# <name>（BusyBox版）

## 名前
<name> - 何をするコマンドか（1行）

## 概要
何が分かる/できるか（2〜5行）

## よく使う
- `...`
- `...`

## 例
### 1) 〜〜〜
`...`

## オプション（要点）
- `-x`: 〜〜〜

## 注意
- BusyBox のビルド設定で挙動が違う場合がある
- UmuOS の想定用途では 〜〜〜

## 関連
- `...`
```

## 持ち込み運用（UmuOS）

### 永続（disk.img）
- バイナリ: `/umu_bin/uman`
- ページ: `/usr/share/uman/ja/*.md`

### 揮発（/run 等）
- `/run/uman/ja/*.md` へ投入
- `UMAN_PATH=/run/uman/ja` で参照

---

## 実装メモ（Cで作る場合）

最小の実装は「ファイルを開いてstdoutへコピー」だけ。

- stdin/stdout はバイナリ透過（UTF-8そのまま）
- パストラバーサル対策
	- `<name>` は `[A-Za-z0-9._-]` 程度に制限
	- `/` や `..` を拒否
- 失敗時の終了コード
	- 使い方間違い: 2
	- ページ無し: 1
	- I/Oエラー: 1

---

## 進め方（提案）

1) ページ置き場の確定（`/usr/share/uman/ja`）
2) `uman` 最小実装（表示のみ）
3) `netstat.md` と `nc.md` を作る（運用に直結）
4) `-k`（検索）を必要になってから追加
