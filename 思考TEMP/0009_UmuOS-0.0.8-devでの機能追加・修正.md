# 0009 UmuOS-0.1.8-devでの機能追加・修正

## ① 修正する項目

- Enterキー問題（CRNL変換の全体対応）
	- 症状: シリアル/コンソールで Enter が期待通りに「次の行」として扱われず、`more` 等で1行送りにならない場合がある
	- 目標: ttyS（getty）でも pts（telnet/login）でも同じ挙動を確保する
	- 方針: 端末セッションの入口で CR→NL を吸収する（OS全体の体験として揃える）
	- 最小実装案（入口で揃える）
		- `umu_getty`（ラッパ）を用意し、対象TTYに `stty icrnl onlcr echo` を当ててから本物の getty を `exec`
			- inittab: `ttyS0/ttyS1` の `getty` 呼び出しを `umu_getty` に差し替える
			- getty が起動するたびに適用される（respawnにも強い）
		- `umu_login`（ラッパ）を用意し、`stdin` が tty のとき `stty icrnl onlcr echo` を当ててから本物の login を `exec`
			- telnetd: `-l /bin/login` を `-l /sbin/umu_login` のように差し替える（pts側の入口を揃える）
		- 保険（任意）: `/etc/profile` に `if [ -t 0 ]; then stty icrnl onlcr echo; fi` を入れて、想定外の入口でも揃える
