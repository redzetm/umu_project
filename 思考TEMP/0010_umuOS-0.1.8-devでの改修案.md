
# UmuOS-0.1.8-dev での改修案（決定事項ログ）

更新日: 2026-03-02

## 決定事項

- `umu_command` 配下のコマンドは **`switch_root` 後（永続 rootfs 側）でのみ利用**する
	- **initramfs には `umu_command` を入れない**
- `switch_root` 前（initramfs / early userspace）は **BusyBox + 固定の `/init`** に収束させ、起動シーケンスを単純化する

## 理由

- `switch_root` 前は「成立条件が多い」領域なので、ツールチェーン/依存/挙動差を増やさず **再現性を最優先**したい
- BusyBox は early userspace に必要な最低限（例：`mount`/`mkdir`/`cat`/`sh`/`switch_root` 等）を **小さく・静的に・一体で**持てるため、initramfs をシンプルに保てる
- `umu_command` はユーザーランド側の研究・拡張のための領域として位置づけ、起動成立（initramfs）と責務分離する
	- `switch_root` 後に `/umu_bin`（`umu_command` 由来）を PATH 最優先にして使う方針にする

## 補足（運用イメージ）

- initramfs（`switch_root` 前）
	- BusyBox を中心に `/init` で必要最小限の初期化を行い、rootfs をマウントして `switch_root` する
- rootfs（`switch_root` 後）
	- `/etc/init.d/rcS` 等で PATH を設定し、`/umu_bin`（= `umu_command` 由来のコマンド群）を優先して利用する
