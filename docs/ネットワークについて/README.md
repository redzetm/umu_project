# ネットワークについて

このディレクトリは、UmuOS におけるネットワーク機能に関する
**設計判断・非目標・思想**を記録するための場所です。

ここに置かれている文書は、実装コードではありません。
また、将来必ず実装されることを約束するものでもありません。

UmuOS は、機能を追加する前に、
「なぜそれを行うのか」「なぜ今は行わないのか」を
明確に言語化することを重視しています。

---

## このディレクトリの目的

- ネットワーク機能に関する **設計上の前提条件** を明示する
- 現時点で **実装していない理由** を記録する
- 将来の実装に対して、誤解や期待のズレを防ぐ
- UmuOS が「何を目指していないか」を明確にする

UmuOS は、実装量よりも **観測可能性・理解可能性** を重視する
研究・検証志向の OS プロジェクトです。
そのため、ネットワーク機能についても、
段階的かつ意図的に扱われます。

---

## 現状の位置づけ

現時点の UmuOS には、以下の特徴があります。

- 常時利用可能な通信機能は存在しない
- セキュリティ用途を目的としたネットワーク機能は提供しない
- インターネット接続を前提としない

※ここでいう「存在しない」は “現時点（導入前）” の話です。
将来は研究目的に限り、閉域で段階的に導入し得ます（例：ロードマップ参照）。

ただし将来、研究目的（観測・比較・理解）のために、
閉域ネットワーク内に限定した通信機能を導入する可能性はあります。
その場合も「実用OSとしての安全なリモート運用」を目的とはせず、
なぜ危険なのか／なぜ暗号化や認証基盤が必要になるのか、を確かめる材料として扱います。

これは「未完成」や「不足」ではなく、
**意図的な設計判断**です。

詳細な理由については、個別の文書を参照してください。

---

## 各文書の概要

### なぜ通信機能がないのか.md

- なぜ UmuOS は最初から通信機能を持たないのか
- OS 内部の健全性を優先する理由
- 「できない」のではなく「まだやらない」という判断

---

### telnetを選ぶ理由.md

- なぜ SSH ではなく TELNET を検討対象とするのか
- 観測性を最優先するという設計思想
- ネットワークを「理解する対象」として扱う理由

※ TELNET を「推奨の運用手段」として扱う意図はありません。
UmuOS では、危険な要素も含めて“学ぶために扱う”という位置づけです。

---

### 将来のネットワーク周りの展望.md

- ネットワーク機能が導入されるとしたら、どの段階か
- TELNET が通過点である理由
- UmuOS におけるネットワークの最終的な位置づけ

## 非目標について

このディレクトリで扱う内容は、以下を目的としていません。

- 実用的なリモートログイン環境の提供
- セキュアな通信基盤の構築
- 一般的な Linux ディストリビューションとの機能比較

※ここでいう非目標は「推奨の運用として提供しない／最終目的にしない」という意味です。
研究のために一時的に実装して観測・比較すること自体を否定しません。

UmuOS は、既存の OS を置き換えることを目的としていません。
理解と観測を通じて、OS の内部構造を明らかにすることを目的としています。
