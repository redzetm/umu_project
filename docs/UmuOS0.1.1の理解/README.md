# UmuOS 0.1.1 の理解について

## このドキュメントの目的

このディレクトリは、UmuOS 0.1.1 を **自分自身が確実に理解するため** に作成したものである。

UmuOS 0.1.1 は、実装としては成功しており、実際に起動し、意図した振る舞いをしている。

しかし、

- なぜこの構成になっているのか
- なぜこの選択をしたのか
- 他の選択肢を取らなかった理由は何か

といった点について、**100% 言語化できているとは言えない**。

このドキュメント群は、その「理解の隙間」を埋めることを目的とする。

---

## 想定している読み手

- 第一の読み手は **未来の自分**
- 数ヶ月後、数年後に読み返しても、
  「ああ、そういう理由だったな」と思い出せることを目指す

他人が読んで参考になるのであれば、それは副産物である。

## UmuOS 0.1.1 に対する現在の認識

現時点での認識は以下の通り。

- 起動シーケンスは把握している
- rcS が何をしているかは「動き」としては理解している
- BusyBox init との違いも感覚的には分かっている

一方で、

- 設計判断を一つ一つ説明できるかというと怪しい
- 「なぜそうしたのか」を即答できない箇所がある
- upstream の思想と自分の思想の境界が曖昧な部分がある

これらを **曖昧なままにしない** ために、文章として残す。

## このディレクトリに含める内容

このディレクトリでは、以下のような内容を扱う。

- UmuOS0.1.1起動の一連の流れの明確化

- init からユーザーランドに至る流れ
- early userspace で何をして、何をしていないか

- その他、理解しておくことが必要な事項
