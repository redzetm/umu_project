# （将来）Rustを試す理由

この文書は、UmuOS において
**将来 Rust 言語を試す可能性がある理由**を整理するためのものです。

ここに書かれている内容は、
Rust の採用を決定したことを意味するものではありません。
また、将来必ず Rust を使うという約束でもありません。

あくまで、
「なぜ Rust が検討対象になり得るのか」
という **判断の前提**を記録するための文書です。

---

## Rust を検討対象とする理由

Rust は、以下のような特徴を持つ言語です。

- メモリ安全性を言語レベルで扱う
- 所有権やライフタイムといった概念を明示的に持つ
- 実行時コストを抑えつつ、安全性を高める設計

これらは、OS の実装や低レイヤのプログラミングにおいて、
無視できない性質です。

UmuOS において Rust を検討対象とする理由は、
これらの特徴そのものではなく、

- **安全性がどのように実現されているのか**
- **その仕組みが OS の挙動理解にどう影響するのか**

を観測・理解する価値があると考えられるためです。

---

## C言語との対比としての価値

現時点で UmuOS は主に C 言語を使用しています。

C 言語は、
OS の内部構造を直接的に観測する上で非常に有効ですが、
同時に多くの責任を実装者に委ねます。

Rust を試すことは、

- その責任の一部を言語が引き受けた場合
- 実装や設計の思考がどのように変化するのか

を比較・観測する機会にもなります。

これは C 言語を否定するためではなく、
**理解を深めるための対比**としての価値です。

---

## 実用性を目的としない

UmuOS において Rust を試す場合でも、

- 安全な OS を作ること
- 実運用に耐える実装を目指すこと

を目的とするわけではありません。

あくまで、

- 言語が設計判断に与える影響
- 抽象化が観測性に与える影響
- 思考と実装の距離の変化

を理解するための試行として位置づけられます。

---

## 試すとした場合の前提

将来 Rust を試す場合でも、

- 一部の機能に限定する
- 観測対象を明確にする
- 判断理由を文書として残す

といった前提を守ることを想定しています。

これは言語選択を「なんとなく」にしないための、
UmuOS における基本姿勢です。

---

## まとめ

Rust を検討対象とする理由は、

- 新しい言語を採用したいからではなく
- 流行しているからでもなく
- C 言語に不満があるからでもありません

OS の挙動や設計判断を、
**別の視点から観測・理解する可能性**があるためです。

この文書は、
その可能性を忘れないための記録として存在します。
