# 言語について

このディレクトリでは、UmuOS における
**プログラミング言語の選択に関する考え方**を記録します。

ここに書かれている内容は、
特定の言語を推奨したり、固定したりするものではありません。

UmuOS がその時点で
「なぜその言語を選んだのか」
「なぜ別の言語を選ばなかったのか」
という **設計判断の背景**を残すことを目的としています。

---

## 言語は目的ではなく道具である

UmuOS において、プログラミング言語は目的ではありません。

- 流行しているかどうか
- 一般的に評価が高いかどうか
- 他の OS が採用しているかどうか

といった理由で言語を選ぶことはありません。

UmuOS では常に、

- OS の挙動を理解できるか
- 内部構造を観測しやすいか
- 思考と実装の距離が近いか

という観点から、その時点で最も適した道具を選びます。

---

## 選択は固定されない

このディレクトリに記録される言語選択は、
将来にわたって固定されるものではありません。

理解が進めば、

- 別の言語を試す
- 一部だけ異なる言語を使う
- 過去の選択を見直す

といった判断が行われる可能性もあります。

それらは設計のブレではなく、
**理解が進んだ結果としての自然な変化**です。

---

## このディレクトリの役割

このディレクトリは、

- 各 UmuOS バージョンの README に
  毎回同じ説明を書かなくて済むようにするため
- 将来の自分が、
  当時の判断理由を正確に思い出せるようにするため
- 言語選択が「なんとなく」にならないようにするため

に存在します。

実装の詳細や使い方ではなく、
**判断の理由と姿勢**を記録する場所です。

---

## 含まれる文書について

- `C言語について.md`  
  現時点で UmuOS が C 言語を選択している理由を記録します。

- `Rustを試す理由.md`（将来）  
  将来、Rust を試す場合に、
  なぜそれを選んだのかを記録するための文書です。

- `言語選択の変遷.md`（将来）  
  言語選択に関する判断の変化を、
  時系列で振り返るための記録です。

---

## まとめ

UmuOS における言語選択は、

- 正解を探すためのものではなく
- 比較表を作るためのものでもなく
- 将来を縛るためのものでもありません

その時点での理解に基づき、
**最も誠実な道具を選んだ結果**を残すためのものです。

このディレクトリは、
その判断を忘れないための記録として存在します。
