# C言語について

UmuOS の実装において、現時点では主に C 言語を使用しています。

この選択は、流行や性能比較の結果ではありません。
また、C 言語を唯一の正解として位置づけるものでもありません。

UmuOS の研究目的に照らしたとき、
**最も思考と実装の距離が近い言語**として、
現時点では C 言語が適していると判断しています。

---

## C言語を選んでいる理由

C 言語は、多くを隠しません。

- メモリを明示的に扱う必要がある
- 失敗は戻り値として現れる
- 抽象化は最小限である
- 実行時の挙動が比較的直接的に追える

これらの性質は、一般的には「不親切」とされることもあります。
しかし UmuOS においては、その不親切さこそが重要です。

OS の内部構造や挙動を理解し、観測するためには、
何が起きているのかを自分の目で追えることが不可欠です。

C 言語は、そのための余計な層をあまり持ち込みません。

---

## Linux カーネルとの距離

UmuOS は Linux カーネルをベースとした OS です。

Linux カーネル自体が C 言語で書かれていることもあり、
C 言語を用いることで、

- カーネルの挙動とユーザーランドの関係
- システムコール境界で起きていること
- メモリや構造体の扱われ方

を、比較的素直に対応づけて理解できます。

これは互換性のためではなく、
**理解のための距離感**として重要だと考えています。

---

## C言語を神格化しない

UmuOS において、C 言語は思想ではありません。

- C でなければならない理由はない
- 将来、別の言語を試す可能性も否定しない
- 一部だけ異なる言語を使うこともあり得る

現時点で C 言語を選んでいるのは、
あくまで「今の理解段階において最も誠実な道具」だからです。

理解が進めば、この判断が変わることも自然なことです。

---

## この選択について

C 言語を選んでいる理由は、
「速いから」でも「一般的だから」でもありません。

- OS の挙動を説明できるか
- 観測した結果を自分の言葉で語れるか
- 実装と理解が乖離しないか

これらを重視した結果として、
現時点では C 言語を選んでいます。

この文書は、
その判断を忘れないための記録として存在します。
