# UmuOS 設計思想　

UmuOS は、日常利用を目的とした OS ではない。  
UmuOS は **研究と観測のための実在する OS** である。

---

## 1. 目的

UmuOS の目的は、  
OS や Linux カーネルの挙動を **理解し、観測し、検証すること** にある。

- 便利さは最優先事項ではない
- 流行や互換性は目的ではない
- 完成を目指すこと自体が目的ではない

「動くこと」よりも  
**「なぜそう動くのかが見えること」** を重視する。

---

## 2. 互換性について

UmuOS は POSIX や GNU との互換性を目標としない。

- 既存ツールの挙動を再現する義務はない
- 一般的な Linux ディストリビューションと同じである必要はない

UmuOS 内で一貫した設計と挙動を持つことを最優先とする。

互換性は、必要になった場合にのみ  
**意図的に導入するもの** とする。

---

## 3. OS と開発環境の分離

UmuOS は **実行環境** であり、  
開発環境を内包することを前提としない。

- 開発は外部環境（例: Ubuntu）で行う
- UmuOS には自己完結したバイナリを持ち込む
- OS 内部は可能な限り単純に保つ

これは制限ではなく、  
**責務分離による設計判断**である。

---

## 4. コマンドとツールの考え方

UmuOS のコマンドは、

- 一般ユーザー向けではない
- 互換ツールの代替ではない
- 研究・観測のための道具である

そのため、

- 名前やオプションは UmuOS 独自でよい
- 出力は機械的でよい
- 使いやすさより理解しやすさを優先する

---

## 5. 完成について

UmuOS は完成を目指さない。

- 未完成でも価値がある
- 途中の設計判断そのものが成果である
- 止まった時点の状態も研究結果である

UmuOS は「完成品」ではなく、  
**思考と試行の記録として存在する OS** である。

---

## 6. この設計思想について

この設計思想は、  
将来の自分が迷ったときに立ち返るためのものである。

新しい機能や変更を加える際は、  
この文章と矛盾していないかを確認する。

---

### 一言で言うと

> **UmuOS は「使うための OS」ではなく、「理解するための OS」である。**
